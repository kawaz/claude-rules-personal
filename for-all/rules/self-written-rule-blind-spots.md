# 自作ルール・self-check の盲点警戒

自分で制定したルール / self-check / DR は **片面しか見ていない**可能性が高い。
制定後に「対極のチェックが漏れていないか」を必ず自問する。

## よくある片面パターン

- 「新規に X を入れるときの check」だけ書いて、「既存 X の不備を発見する check」が漏れている
- 「禁止事項」だけ書いて、「禁止違反を検出する仕組み」が漏れている
- 「追加時の整合確認」だけ書いて、「削除時の整合確認」が漏れている
- 「成功時のログ」だけ書いて、「失敗時の検知」が漏れている

## How to apply

- ルール / DR / self-check を起草したら、最後に「対極の側面」を 1 個以上書き出して確認
- 既存ルールを読んだとき、対極が抜けていれば指摘または補完
- check list を作るときは「入れる側 / 出す側」「成功 / 失敗」「新規 / 既存」の両方を含める

## Why

自分でルールを書く瞬間は特定の文脈 (= 直近の失敗) に強く影響されている。
その文脈の対極にあるケースは見えにくい。意識的に対極を探さないと埋もれる。
