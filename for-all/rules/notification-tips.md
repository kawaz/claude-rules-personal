# 通知系 tips (say / 1Password エラー)

## `say` への英略語はカタカナで渡す

`say` (macOS 音声合成) に英大文字略語をそのまま渡すと変な読み方になる
(例: `OTP` → 「おっとぴー」)。**頭字語・略語はカタカナに変換してから渡す**。

| 元 | 渡す文字列 |
|---|---|
| OTP | オーティーピー |
| URL | ユーアールエル |
| API | エーピーアイ |
| CI | シーアイ |
| PR | ピーアール |
| 1Password | ワンパスワード |
| jj | ジェイジェイ |

- 対象は頭字語・略語 (各文字を順に読むもの) のみ。普通の英単語
  (`hello`, `error`, `failed` 等) は say が正しく読めるのでそのままで良い
- 判断に迷う固有名詞 (プロダクト名、ツール名) はカタカナ化を優先 (誤読リスク回避)
- 文中の日本語と混在してよい。カタカナ化が必要なのは頭字語の部分だけ
  (例: `say "シーアイ が failed しました"`)

## 1Password エラー時の対応

git / jj / ssh の署名操作は通常タイムアウトしない。`1Password: failed to
fill whole buffer` エラーが発生した場合のみ:

1. 音声通知: `say 'ワンパスワードのエラーで困ってます。気づいて！'`
2. ユーザー在席と判断できる場合のみ AskUserQuestion で再試行を確認
   (選択肢: 「再試行する (Recommended)」/「キャンセル」)
3. 不在推定時 (「離席する」「寝る」等の直前発言 / /remote-control 中 /
   フルオートモード) は QA を実行せず、音声通知のみで push をスキップして
   他の作業を続行

jj 管理リポでは `git.sign-on-push` により `jj git push` 時にまとめて署名する
ため、commit 時の 1Password エラーは無視し、push 時のみ上記対応。
