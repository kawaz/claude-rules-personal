# `say` コマンドへの英略語はカタカナで渡す

`say` (macOS の音声合成) に英大文字略語をそのまま渡すと、変な読み方になる
(例: `OTP` → 「おっとぴー」「オトプ」)。**頭字語・略語はカタカナに変換してから渡す**。

## 変換例

| 元 | 渡す文字列 |
|---|---|
| OTP | オーティーピー |
| URL | ユーアールエル |
| API | エーピーアイ |
| CI | シーアイ |
| PR | ピーアール |
| 1Password | ワンパスワード |
| jj | ジェイジェイ |

## 対象の判断

- **頭字語・略語** (各文字を順に読むもの): カタカナ化
- **普通の英単語** (`hello`, `error`, `failed` 等): そのままでも `say` が正しく読める
- **判断に迷う固有名詞** (プロダクト名、ツール名): カタカナ化を優先 (誤読リスク回避)

## 例

```bash
# Bad
say "OTPの入力をお願いします"
say "CI が failed しました"

# Good
say "オーティーピーの入力をお願いします"
say "シーアイ が failed しました"
```

文中の日本語と混在してよい (英単語部分は say がそのまま読める)。**カタカナ化が必要なのは頭字語の部分だけ**。
