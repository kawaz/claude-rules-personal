# ルール記述ガイドライン

ルールファイル作成時は以下に従うこと：

1. **具体的**: 手順が決まっている場合はコマンドをそのまま記載
2. **完結**: 他ファイル参照なしで理解可能
3. **省コンテキスト**: 背景説明を最小限に、行動指針を明確に

理由: ルールは毎回コンテキストに読み込まれるため、冗長な記述はトークンを浪費する。

## 例

```markdown
# プロジェクトXのビルド手順

ビルド: `cargo build --release`
チェック: `cargo fmt --check && cargo clippy -- -D warnings`
テスト: `cargo test`
```
