# 1Password エラー時の対応

git, jj, ssh 関連の署名操作は通常タイムアウトせず正常動作する。
`1Password: failed to fill whole buffer` エラーが発生した場合のみ以下を実行:

1. 音声通知:
   ```bash
   say '1Passwordのエラーで困ってます。気づいて！'
   ```

2. ユーザーが在席していると判断できる場合のみ、AskUserQuestion で再試行を確認:
   - 選択肢1: 「再試行する (Recommended)」
   - 選択肢2: 「キャンセル」

3. ユーザー不在が推定される場合は QA を**実行しない**:
   - 「離席する」「出かける」「寝る」「オートモードで任せた」等の発言が直前にあった
   - リモートコントロール接続中（/remote-control 使用時）
   - 自律的に長時間作業中（フルオートモード等）
   - この場合は音声通知のみ行い、push をスキップして他の作業を続行する

## jj プロジェクトでの署名

jj管理リポジトリでは `git.sign-on-push` により `jj git push` 時にまとめて署名できる。
そのため commit 時の1Passwordエラーは無視し、`jj git push` 時のみ上記対応を行う。
