# 作業原則

## 指示通り作業（スキップ禁止）

- 読めと言われたファイルは全て読み、理解する
- やれと言われたら手順通り全部やる
- 「時間節約」等の理由で手順・読込をスキップしない
- 迷ったらスキップせず実行、またはユーザーに確認

## TODOファースト

- 複数ステップの作業は開始前に一連のTODOを作成
- `addBlockedBy`で依存関係を設定しワークフロー構築
- 「{次に/後で/終わったら}XXする」は思いついた時点でTODO作成

## サブエージェント活用

- 分析・実行などの実作業はサブエージェントに委譲し、メインは設計・タスク分解・成果の監査・統合・ユーザ対話に徹するのが基本
- 結果がすぐ不要ならバックグラウンド実行
- バックグラウンドタスク＋条件付きTODOで効率的に並列化

委譲の強度は **メインモデルの tier で変わる**。詳細は [[top-tier-model-delegation]]:

- **最上位 tier (Fable 等) メイン時**: 実作業は基本的にサブエージェントへ。メイン直は下記「メインが直接行ってよい作業」のリストに限る
- **中位 tier (Opus / Sonnet 等) メイン時**: メイン直の許容範囲が広い。下記リストに加え、委譲のほうがコストが大きい場合 (例: 1 ファイル内定型修正 / 調査なしで完結する数行変更 / 起動説明の方が作業量を上回る) もメイン直で OK

### メインが直接行ってよい作業 (tier 問わず)

- 単純なコマンド（git status等）
- ユーザーへの質問・報告
- TODOリスト操作
- 軽微編集 (= 数行修正 / 小ファイル 1 個 / one-shot script)
- 設計と実装が不可分な箇所 (= 設計判断しながら書く必要があるもの)

判定は最終的に「メインモデル自身の感覚」に従う。下位モデルが固定の数値基準を
書いて最上位 tier の判断を縛るのは越権 (= 最上位 tier の dogfooding で見えた
場面に応じて kawaz が随時調整する)。**迷ったら委譲** を safety net に。

### 理由

- メインコンテキスト節約で長期セッション維持
- ユーザーチャット優先（長時間作業を同期的に待つと追加指示不可）
- 最上位 tier メインはサブエージェント分も含めると支配的コストになる
  ([[top-tier-model-delegation]] の Why 参照)

### サブエージェントとの入出力（重要）

**`Full transcript available at:`のファイルは絶対に読んではいけない**

- 起動時の`output_file:`のパス
- 完了時の`<task-notification>`と一緒に通知される`Full transcript available at:`のパス

これらのファイルにはJSONL形式で全会話履歴（Read/Bashなど全ツールの入出力）が含まれ、メインコンテキストが溢れて動作不能になる。

サブエージェントの最終応答は`<task-notification>`の`<result>`タグ内に全文含まれるため、それを使用する。
